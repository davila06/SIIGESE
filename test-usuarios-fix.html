🔧 Fix Implementado: Campo Nombre de Usuario

Resolución del problema: "el campo nombre de usuario en el módulo de usuarios no carga al seleccionar un registro"

✅ Problema Resuelto

Issue: El campo "nombre de usuario" (userName) no se cargaba al seleccionar un registro en el módulo de usuarios para edición.

Causa Raíz: Los datos de usuarios en el mock API no tenían el campo userName definido.

🛠️ Fix Implementado

Se agregaron los campos faltantes a la estructura de usuarios en mock-api.interceptor.ts:

private users = [
  {
    id: 1,
    userName: 'admin',
    email: 'admin@sinseg.com',
    firstName: 'Admin',
    lastName: 'SINSEG',
    isActive: true,
    roles: [...],
    createdAt: new Date(),
    lastLoginAt: new Date()
  }
]

Se agregaron los siguientes campos a todos los usuarios:

  • userName: Campo que faltaba y causaba el problema
  • isActive: Estado activo del usuario
  • createdAt: Fecha de creación
  • lastLoginAt: Última fecha de login

  1. Acceder a la aplicación: Abrir Aplicación
