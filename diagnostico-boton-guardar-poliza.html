🔧 Diagnóstico: Botón Guardar Póliza Deshabilitado

Herramienta de diagnóstico para identificar por qué el botón "Guardar Póliza" está siempre deshabilitado

🔍 Problema Identificado

⚠️ Síntoma: El botón "Guardar Póliza" aparece siempre deshabilitado (gris) y no permite hacer click, independientemente de si los campos están llenos o no.

📋 Código Actual del Botón

<button mat-raised-button color="primary" type="submit" [disabled]="polizaForm.invalid || isLoading"> <mat-icon>{{isEditMode ? 'update' : 'save'}}</mat-icon> {{submitButtonText}} </button>
🎯 Análisis: El botón se deshabilita cuando polizaForm.invalid es true o cuando isLoading es true.

🚨 Posibles Causas del Problema

1. 📝 Validaciones de Campos Requeridos

numeroPoliza: required, maxLength(50) REQUERIDO
modalidad: required, maxLength(100) REQUERIDO
nombreAsegurado: required, maxLength(200) REQUERIDO
prima: required, min(0) REQUERIDO
fechaVigencia: required REQUERIDO
frecuencia: required, maxLength(50) REQUERIDO
aseguradora: required, maxLength(100) REQUERIDO

2. 🔄 Estado de Carga (isLoading)

Si isLoading está en true, el botón se mantiene deshabilitado.

3. 📅 Problema con Campo de Fecha

Sospecha Principal: El campo fechaVigencia podría tener problemas de formato o inicialización que mantienen el formulario inválido.

🧪 Código de Diagnóstico para Consola

Copia y pega este código en la consola del navegador (F12) cuando estés en la página de pólizas:

// 🔍 DIAGNÓSTICO COMPLETO DEL FORMULARIO DE PÓLIZAS console.log('🔧 === DIAGNÓSTICO DEL BOTÓN GUARDAR PÓLIZA ==='); // Buscar el componente de pólizas const polizasComponent = ng.getOwningComponent(document.querySelector('app-polizas')); if (polizasComponent) { console.log('✅ Componente encontrado:', polizasComponent); // Verificar estado del formulario const form = polizasComponent.polizaForm; console.log('📝 Estado del formulario:'); console.log('- Válido:', form.valid); console.log('- Inválido:', form.invalid); console.log('- Tocado:', form.touched); console.log('- Sucio:', form.dirty); // Verificar cada campo individual console.log('\n📋 Estado de cada campo:'); Object.keys(form.controls).forEach(key => { const control = form.get(key); console.log(`${key}:`, { value: control.value, valid: control.valid, invalid: control.invalid, errors: control.errors, touched: control.touched, dirty: control.dirty }); }); // Verificar estado de loading console.log('\n🔄 Estado de carga:', polizasComponent.isLoading); // Verificar modo de edición console.log('✏️ Modo edición:', polizasComponent.isEditMode); // Verificar botón específicamente const submitButton = document.querySelector('button[type="submit"]'); if (submitButton) { console.log('\n🔘 Estado del botón:'); console.log('- Deshabilitado:', submitButton.disabled); console.log('- Clase disabled:', submitButton.classList.contains('mat-button-disabled')); } // Mostrar valores del formulario console.log('\n📊 Valores actuales del formulario:', form.value); } else { console.log('❌ No se encontró el componente de pólizas'); }

Solución 2: 🔧 Validación Contextual Similar a Usuarios

// Crear método similar al isFormValidForMode() de usuarios isFormValidForSubmission(): boolean { const requiredFields = ['numeroPoliza', 'modalidad', 'nombreAsegurado', 'prima', 'fechaVigencia', 'frecuencia', 'aseguradora']; return requiredFields.every(field => { const control = this.polizaForm.get(field); return control && control.valid && control.value; }); } // Y cambiar el botón a: [disabled]="!isFormValidForSubmission() || isLoading"
